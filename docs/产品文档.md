## 1. 背景与问题

姐姐目前在**闲鱼**上卖衣服，规模约**几十件 SKU**，后续可能拓展到**微信、抖音**等渠道。

当前痛点：

- 商品分散在多个闲鱼号，难以统一看清库存与在售情况
- 衣服来源有两种：**自己以前买的（相当于零成本）** 和 **后来进货的（有进价）**，混在一起不好算账
- 成交记录靠记忆或零散表格，**利润、销售额、哪个号卖得好** 算不清楚
- 闲鱼没有开放商家 API，无法自动同步订单，需要一套**手动但高效**的管理工具

---

## 2. 产品目标

### 2.1 核心目标

1. **管货**：统一管理商品、分类、库存、图片、闲鱼链接
2. **管账**：清晰记录每笔成交，自动计算利润
3. **管号**：支持多个闲鱼账号，对比各账号经营表现
4. **辅助决策**：通过数据概览和账号分析，知道哪个号做得好、哪里需要改进

### 2.2 非目标（第一期不做）

- 不替代闲鱼 App 本身（聊天、发货仍在闲鱼完成）
- 不对接闲鱼官方订单 API（平台未开放）
- 不做完整 ERP / 财务系统
- 不做小程序商城（预留架构，第二期再议）

## 4. 业务场景

### 场景 A：自有闲置衣服

- 姐姐以前自己买的衣服，在闲鱼上清库存
- **进价记为 ¥0**
- 卖出后利润 ≈ 成交额 − 闲鱼服务费 − 运费 − 其他费用

### 场景 B：进货转卖

- 从批发市场/厂家拿货
- 需记录**进价**和**售价**
- 利润 = 成交额 − 进价 × 数量 − 各项费用

### 场景 C：多闲鱼号并行

- 姐姐有多个闲鱼账号同时在卖
- 不同号可能卖不同品类或承担不同角色（主号 / 清仓号等）
- 需要知道：**哪个号卖得多、哪个号利润高、哪个号值得投流量**

### 场景 D：闲鱼成交后记账

- 在闲鱼 App 内成交
- 回到管理后台**手动录入销售**（或批量 Excel 导入）
- 系统自动扣库存、算利润、归入对应闲鱼号

## 6. 功能详述

### 6.1 数据概览

**目的**：打开系统第一眼掌握整体经营状况。

| 模块 | 内容 |
|------|------|
| 核心指标卡片 | 商品数、SKU/库存、库存货值（按进价）、本月净利润 |
| 分析概览 | 按 本周/上周/本月/上月/自定义 查看卖出件数、成交额、总利润（图表） |
| 闲鱼账号概览 | 各账号销售额、利润、在售/下架商品数、流量投入、净收益等 |
| 最近销售 | 最新成交记录 |
| 低库存预警 | 库存 ≤ 预警值的 SKU |

---

### 6.2 商品管理

**目的**：统一管理所有在售商品。

#### 商品字段

| 字段 | 说明 |
|------|------|
| 商品名称 | 必填 |
| 分类 | 上衣 / 裤子 / 裙子 / 外套 / 配饰等 |
| 商品来源 | **自有闲置** 或 **进货** |
| 归属闲鱼号 | 该商品主要在哪个号售卖 |
| 闲鱼链接 | 可选，方便跳转 |
| 货号 | 可选 |
| 状态 | 在售 / 主动下架 / 售出下架 |
| 商品图片 | 支持上传至 Supabase Storage |
| 备注 | 可写闲鱼标题、卖点等 |

#### SKU（规格）

每件商品可有多条 SKU，按 **颜色 + 尺码** 区分：

| 字段 | 自有闲置 | 进货 |
|------|----------|------|
| 进价 | 固定 ¥0（不可改） | 必填 |
| 售价 | 建议售价 | 建议售价 |
| 库存 | 当前件数 | 当前件数 |
| 低库存预警 | 默认 2 件 | 默认 2 件 |

#### 列表能力

- 搜索：名称 / 货号
- 筛选：分类、状态、来源
- 分页

---

### 6.3 闲鱼导入

**目的**：减少手工录入，从闲鱼分享链接快速创建商品。

流程：

1. 粘贴闲鱼分享文案或链接
2. 系统解析商品名、价格、图片、链接等
3. 补充来源、分类、归属账号、进价等
4. 保存为商品（可选同时记一笔销售）

---

### 6.4 库存管理

**目的**：随时掌握还有多少货、货值多少。

| 功能 | 说明 |
|------|------|
| 库存列表 | 按商品/SKU 展示进价、售价、库存、货值 |
| 库存调整 | 入库 / 出库 / 直接调整，并写变动记录 |
| 低库存筛选 | 快速找到快卖完的款 |
| 变动记录 | 可追溯每次库存变化 |

**说明**：销售录入成功后，库存会自动扣减，无需重复操作。

---

### 6.5 销售记录

**目的**：记录每一笔成交，作为利润统计的数据源。

#### 录入方式

1. **手动录入**（最常用）
2. **Excel 批量导入**（下载模板 → 填写 → 上传）

#### 销售字段

| 字段 | 说明 |
|------|------|
| 卖出商品（SKU） | 必选 |
| 销售渠道 | 闲鱼 / 微信 / 抖音 / 其他 |
| 成交闲鱼号 | 闲鱼渠道**必选** |
| 成交日期 | 默认今天 |
| 数量 | 默认 1 |
| 成交额 | 实际收到金额 |
| 单件进价 | 自有闲置自动为 0 |
| 平台服务费 | 闲鱼默认按 0.6% 估算，可改 |
| 运费 / 其他费用 | 可选 |
| 订单状态 | 成功 / 有退款 |
| 备注 | 买家昵称、订单备注等 |

#### 利润计算规则

```text
成功订单：
  利润 = 成交额 − 进价 × 数量 − 平台费 − 运费 − 其他费用

有退款订单：
  利润 = 0
```

#### 其他规则

- 录入成功后**自动扣减库存**
- 删除销售记录**不会**自动恢复库存（防误操作）
- 列表支持按渠道、闲鱼号、关键词筛选

---

### 6.6 闲鱼账号管理

**目的**：管理姐姐的多个闲鱼号。

| 字段 | 说明 |
|------|------|
| 账号名称 | 如「闲鱼主号」「清仓2号」 |
| 平台 | 默认闲鱼（预留微信/抖音） |
| 状态 | 启用 / 停用 |
| 备注 | 卖什么类型、负责什么 |

#### 流量投入记录

每个账号可单独记录**买曝光/推广**等投入：

| 字段 | 说明 |
|------|------|
| 投入日期 | |
| 投入金额 | |
| 备注 | 如「曝光推广」 |

用于账号分析中计算**净收益**（利润 − 流量投入 − 进货成本等）。

---

### 6.7 账号分析

**目的**：对比各闲鱼号表现，找出优势与不足。

#### 统计维度（可按时段筛选）

| 指标 | 说明 |
|------|------|
| 成交单数 | |
| 卖出件数 | |
| 成交额 | |
| 利润 | 已排除退款订单 |
| 利润率 | |
| 客单价 | |
| 流量投入 | |
| 净收益 | 利润 − 投入 − 相关成本 |

#### 智能评价（自动）

| 标签类型 | 示例 |
|----------|------|
| 正向 | 利润冠军、成交额最高、出单活跃、利润率高 |
| 需关注 | 出单偏少、利润偏低 |

#### 改进建议（自动）

例如：

- 「出单量低于平均水平，建议增加曝光或调整定价」
- 「利润率偏低，可适当提高售价或优先卖自有闲置款」
- 「客单价较低，可尝试搭配销售」

---

### 6.8 分类管理

维护商品分类（上衣、裤子、裙子等），支持排序。

## 7. 核心业务流程

### 7.1 新商品上架（进货款）

```text
商品管理 → 新增
  → 来源选「进货」
  → 填写名称、分类、归属闲鱼号
  → 添加 SKU（颜色/尺码/进价/售价/库存）
  → 上传图片
  → 保存
```

### 7.2 自有闲置上架

```text
闲鱼导入（或商品管理 → 新增）
  → 来源选「自有闲置」
  → 进价自动为 0
  → 填售价和库存
  → 保存
```

### 7.3 闲鱼卖出后记账

```text
销售记录 → 录入销售
  → 选择 SKU
  → 选择成交闲鱼号
  → 填成交额（平台费自动估算）
  → 保存
  → 系统自动：扣库存 + 计入利润 + 归入该账号统计
```

### 7.4 月度复盘

```text
数据概览 → 看本月净利润
账号分析 → 切换「本月」→ 对比各号
  → 看哪个号利润最高
  → 看改进建议
  → 决定是否给某号加流量投入
```

## 8. 数据模型（概要）

```text
categories              分类
seller_accounts         售卖账号（闲鱼号等）
account_traffic_investments   账号流量投入

products (SPU)          商品
  ├── source_type       own | purchase
  ├── account_id        归属闲鱼号
  ├── xianyu_link       闲鱼链接
  └── status            active | inactive | sold_out

product_skus            SKU（颜色+尺码+进价+售价+库存）
stock_logs              库存变动
sales                   销售记录
  ├── account_id        成交账号
  ├── order_status      success | refunded
  └── profit            自动计算
```

## 10. 技术方案

| 层级 | 技术 |
|------|------|
| 管理后台 | Vue 3 + Vite + Element Plus + Pinia + ECharts |
| 后端 | Supabase（PostgreSQL + Auth + Storage） |
| 共享类型 | `shared/types.ts`（后期小程序复用） |
| 部署 | 前端 Vercel / 后端 Supabase Cloud |

**选型理由（给维护者）**：

- 前端主导，Supabase 省去自建后端和运维
- 几十 SKU 规模，当前架构足够且轻量
- 同一套数据库，后期接 uni-app 小程序成本低

## 12. 已知限制

| 限制 | 说明 |
|------|------|
| 闲鱼无 API | 订单、聊天无法自动同步，需手动录入或 Excel 导入 |
| 删除销售不恢复库存 | 有意设计，避免误删导致库存虚增 |
| 库存货值 | 仅统计进货款（自有闲置进价为 0，不计入货值） |
| 单人使用 | 当前为姐姐 + 维护者使用，未做多角色权限 |
