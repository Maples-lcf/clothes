| 项目 | 说明 |
|------|------|
| 文档版本 | v1.0 |
| 更新日期 | 2026-06-24 |
| 目标 | 姐姐在闲鱼操作后，用手机 30 秒内同步到系统 |

---

## 1. 姐姐的真实操作链路

```text
闲鱼 App 编辑/发布
        ↓
  需要同步到系统（当前痛点：要开电脑、进后台、找页面）
        ↓
  理想：掏出手机 → 点一下 → 填完 → 完成
```

三个高频动作：

| 动作 | 闲鱼侧 | 系统侧要做的事 |
|------|--------|----------------|
| **上新** | 刚发布/重新上架 | 新建商品 + 拍照 + 标记上新 |
| **售出下架** | 买家付款、商品卖出 | 记一笔销售 + 商品改「售出下架」+ 扣库存 |
| **主动下架** | 暂时不想卖了 | 商品改「主动下架」，库存保留 |

## 2. 推荐路线：H5 先行 → 小程序演进

```text
第二期 A（现在）   mobile/ H5 随手记
                   微信里打开链接就能用，支持拍照上传
                   与 admin 共用 Supabase，数据实时同步

第二期 B（下一步） 闲衣小筑 微信小程序
                   uni-app 复用 shared/ + 业务逻辑
                   增加商品展示、分享、收藏

第三期            小程序内下单 + 微信支付
                   私域卖货，不依赖闲鱼
```

### 为什么先做 H5，不直接做小程序？

| 对比 | H5 随手记 | 微信小程序 |
|------|-----------|------------|
| 开发周期 | 1–2 天可上线 | 需注册主体、审核、发版 |
| 拍照上传 | 浏览器/微信内直接调起相机 | 同样支持 |
| 分享 | 链接发微信 | 原生分享卡片 |
| 与后台关系 | 同一套 Supabase | 同一套 Supabase |
| 后期迁移 | uni-app 可 80% 复用代码 | — |

**结论**：H5 是小程序的「轻量预演」，验证流程后再做「闲衣小筑」正式小程序。

## 3. H5 随手记 · 产品结构

### 3.1 首页（三个大按钮）

```text
┌─────────────────────────┐
│      衣服管家 · 随手记    │
│                         │
│  ┌───────────────────┐  │
│  │  📦 上新           │  │  → 拍图 + 填信息 → 商品管理
│  └───────────────────┘  │
│  ┌───────────────────┐  │
│  │  ✅ 售出下架        │  │  → 选商品 + 填成交价 → 销售记录
│  └───────────────────┘  │
│  ┌───────────────────┐  │
│  │  ⏸ 主动下架        │  │  → 选在售商品 → 改状态
│  └───────────────────┘  │
│                         │
│  本月已售 12 件 · 利润 ¥680 │
└─────────────────────────┘
```

### 3.2 上新页

| 字段 | 说明 |
|------|------|
| 拍照/相册 | 直接调起相机，上传 Supabase Storage |
| 商品名称 | 必填 |
| 售价 | 必填 |
| 来源 | 自有闲置 / 进货（进货显示进价） |
| 闲鱼账号 | 在哪个号卖的 |
| 分类 | 下拉选择 |
| 颜色/尺码 | 默认均色/均码，可改 |
| 库存 | 默认 1 |
| 标记上新 | 默认勾选，填上架日期 |

保存 → `products` + `product_skus`，状态「在售」。

### 3.3 售出下架页

| 步骤 | 说明 |
|------|------|
| 1. 选商品 | 从「在售」列表搜索/点选 |
| 2. 选规格 | 多 SKU 时选颜色尺码 |
| 3. 填成交信息 | 实际成交价、运费、平台费（闲鱼自动 0.6%）、闲鱼账号 |
| 4. 确认 | 创建 `sales` + 商品改 `sold_out` |

### 3.4 主动下架页

| 步骤 | 说明 |
|------|------|
| 1. 选商品 | 从「在售」列表多选 |
| 2. 确认下架 | 批量改状态为 `inactive` |

```text
                    Supabase
                       │
        ┌──────────────┼──────────────┐
        │              │              │
    admin/          mobile/      闲衣小筑(后期)
   管理后台          H5随手记        展示+售卖
        │              │              │
   完整 CRUD        3 个快捷动作     只读商品 + 下单

```

**数据实时同步**：姐姐在 H5 上「售出下架」，后台「销售记录」和「数据概览」立刻更新。

## 5. 闲衣小筑小程序 · 远期规划

### 5.1 定位

面向买家的**私域小店**，与闲鱼互补：

- 闲鱼：公域流量、聊天成交
- 闲衣小筑：朋友圈/微信群分享、固定粉丝复购

### 5.2 功能分期

| 阶段 | 功能 |
|------|------|
| v0.1 | 商品列表、详情、上新标签、联系卖家（跳转微信） |
| v0.2 | 分类筛选、搜索、分享卡片 |
| v1.0 | 购物车、下单、微信支付 |
| v1.1 | 订单管理、发货通知 |

### 5.3 技术选型

```text
uni-app (Vue 3)
  ├── 复用 shared/types.ts
  ├── 复用 Supabase 客户端
  ├── 复用 mobile/ 里的 services 层
  └── 新增 pages/shop/ 买家端页面
```

RLS 已预留：`anon` 角色可读 `status = active` 的商品，小程序展示无需改后端。

## 6. 部署与使用

### H5 开发

```bash
cd mobile
cp ../admin/.env.local .env.local   # 共用 Supabase 配置
npm install
npm run dev                         # http://localhost:5174
```

### 给姐姐用

1. H5 部署到 Vercel（与 admin 同样方式）
2. 把链接**发到微信**或**添加到主屏幕**
3. 建议保存为「衣服管家」快捷方式，和闲鱼 App 放一起

### 微信内打开

- 支持拍照（`<input capture="environment">`）
- 支持长按识别（后期可加）
- 需 HTTPS（Vercel 默认满足）

## 7. 操作对比：改前 vs 改后

| 场景 | 改前（电脑后台） | 改后（H5 随手记） |
|------|------------------|-------------------|
| 闲鱼上新后同步 | 开电脑 → 商品管理 → 新增 → 填表 | 手机 → 上新 → 拍照 → 保存 |
| 卖出记账 | 销售记录 → 录入 → 选 SKU | 售出下架 → 点商品 → 填价格 |
| 暂时下架 | 商品管理 → 搜索 → 编辑改状态 | 主动下架 → 勾选 → 确认 |

预计单次操作从 **3–5 分钟** 缩短到 **30 秒–1 分钟**。
